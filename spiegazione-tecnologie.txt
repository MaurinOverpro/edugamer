SPIEGAZIONE DELLE TECNOLOGIE DI EDUGAMER
=========================================


HTML — IL PUNTO DI PARTENZA
-----------------------------
HTML descrive "cosa c'è" nella pagina:

    <button>Clicca qui</button>
    <p>Ciao studente!</p>

Ma l'HTML puro è statico: non cambia, non reagisce, non calcola nulla.
Per tutto il resto servono gli altri strumenti.


JAVASCRIPT (JS) — IL CERVELLO
-------------------------------
JS è il linguaggio di programmazione del browser. Serve per far fare cose alla pagina:

    // Quando clicchi il bottone, aggiungi 10 XP
    button.addEventListener("click", function() {
        EduGamer.addXP(10);
    });

Tutto il comportamento di EduGamer è JS: le chiamate a Gemini, il calcolo del livello,
il salvataggio nel localStorage... Senza JS la pagina sarebbe solo testo e immagini ferme.

In pratica: HTML dice "cosa mostrare", JS dice "cosa fare".


TAILWIND CSS — IL VESTITO
--------------------------
CSS serve per definire come appare la pagina (colori, dimensioni, posizioni).
Tailwind CSS è un sistema che ti evita di scrivere CSS a mano — usi invece delle
classi già pronte direttamente nell'HTML:

    <!-- Senza Tailwind, dovresti scrivere CSS separato -->
    <button style="background: green; color: white; padding: 10px; border-radius: 8px;">
        Clicca
    </button>

    <!-- Con Tailwind, usi classi con nomi intuitivi -->
    <button className="bg-green-500 text-white p-3 rounded-lg">
        Clicca
    </button>

Ogni parola nella classe fa una cosa precisa:
  - bg-green-500  → sfondo verde
  - text-white    → testo bianco
  - p-3           → padding (spazio interno)
  - rounded-lg    → angoli arrotondati

In EduGamer tutti i colori, le card, i bottoni, i layout sono fatti così.
Per questo nei file vedi elementi con classi lunghissime — è Tailwind.


REACT — IL COSTRUTTORE DINAMICO
---------------------------------
Il problema dell'HTML normale:
  Se l'utente guadagna XP, come aggiorni il numero mostrato a schermo?

Con HTML puro dovresti ricaricare la pagina. Con JS puro dovresti trovare
l'elemento e modificarlo a mano ogni volta.

React risolve questo con un'idea semplice:
  "Descrivi come deve apparire la pagina in base ai dati,
   e React aggiorna tutto da solo."

    function BarraXP({ xp }) {
        return (
            <div className="bg-gray-700 rounded-full">
                <div style={{ width: xp + "%" }} className="bg-green-500">
                    {xp} XP
                </div>
            </div>
        );
    }

Quando "xp" cambia, React ridisegna automaticamente quella parte della pagina.
Non devi fare nulla.

I COMPONENTI DI REACT
React ti fa costruire la pagina a blocchi riutilizzabili chiamati componenti.
In EduGamer ci sono componenti come BarraXP, CardModulo, ChatMessage...
ognuno è una funzione JS che restituisce HTML.

    App
    ├── Header (barra in alto con XP)
    ├── GrigliaModuli
    │     ├── CardModulo (Lavagna)
    │     ├── CardModulo (Matematica)
    │     ├── CardModulo (Italiano)
    │     └── CardModulo (Tutor)
    └── ModalImpostazioni (la finestra API key)


BABEL — IL TRADUTTORE
-----------------------
React usa una sintassi speciale chiamata JSX — è quell'HTML "strano" che trovi
dentro le funzioni JS:

    // Questo NON è HTML normale, è JSX
    return (
        <div className="card">
            <h2>{titolo}</h2>
            <p>{descrizione}</p>
        </div>
    );

Il browser non capisce JSX. Babel è un traduttore che lo converte in JS normale
prima che il browser lo esegua:

    // Babel lo trasforma in questo (internamente):
    return React.createElement("div", { className: "card" },
        React.createElement("h2", null, titolo),
        React.createElement("p", null, descrizione)
    );

Tu non vedi mai questa trasformazione — succede in automatico.
Il type="text/babel" nei tag <script> dei tuoi file è il segnale che dice
al browser: "aspetta, prima passa questo codice a Babel".


COME LAVORANO INSIEME IN EDUGAMER
-----------------------------------

    Tu apri tutor.html
            |
            v
    Il browser carica Tailwind, React, Babel da internet (CDN)
            |
            v
    Babel legge il codice JSX dentro <script type="text/babel">
    e lo traduce in JS normale
            |
            v
    React costruisce l'interfaccia e la inserisce nel <div id="root">
    (tutti i colori/layout vengono da Tailwind)
            |
            v
    Quando scrivi un messaggio, JS chiama Gemini API
            |
            v
    La risposta arriva → React aggiorna solo la parte di chat
    (senza ricaricare la pagina)
            |
            v
    game-system.js aggiunge XP → React aggiorna la barra XP


SCHEMA RIASSUNTIVO
-------------------

    Strumento      Ruolo                        Analogia
    ----------     --------------------------   ---------------------
    HTML           Struttura della pagina       Lo scheletro
    CSS/Tailwind   Aspetto visivo               I vestiti
    JavaScript     Logica e comportamento       Il cervello
    React          UI dinamica e reattiva       Il regista che aggiorna la scena
    Babel          Traduttore JSX > JS          Un interprete simultaneo


STRUTTURA DEI FILE DI EDUGAMER
--------------------------------

    edugamer/
    ├── index.html          <- la "home", da cui si apre tutto
    ├── game-system.js      <- il motore dei punti/XP (condiviso da tutti i moduli)
    ├── lavagna.html        <- modulo Lavagna (mappe mentali con AI)
    ├── matematica.html     <- modulo Matematica (blocchi visivi)
    ├── italiano.html       <- modulo Italiano (correzione testi)
    ├── tutor.html          <- modulo Tutor AI (chat + quiz)
    ├── profilo.html        <- dashboard profilo (XP, livelli, achievement)
    ├── discover.html       <- metodo DISCOVER
    └── risolvitore.html    <- risolvitore problemi

Ogni file HTML contiene tutto insieme: struttura, stile e logica.
Sono moduli autonomi collegati dalla home (index.html) e dal file game-system.js.


COME VIENE SALVATO IL PROGRESSO
---------------------------------
I dati vengono salvati nel localStorage del browser (una piccola memoria locale).
Non escono mai dal computer. Le chiavi principali sono:

    edugamer_stats        -> XP, streak, contatori azioni
    edugamer_achievements -> achievement sbloccati
    gemini_api_key        -> la chiave API inserita nelle impostazioni


COME FUNZIONA L'AI (GEMINI)
-----------------------------

    Tu scrivi/carichi qualcosa
            |
            v
    Il codice JS prepara un "prompt" (istruzioni per l'AI)
            |
            v
    Viene inviata una richiesta a Google (tramite internet)
            |
            v
    Google risponde con il testo generato
            |
            v
    React aggiorna l'interfaccia con la risposta
